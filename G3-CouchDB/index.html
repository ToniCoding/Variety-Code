CouchDB: Un gestor de bases de datos

CouchDB es un gestor de bases de datos de código abierto, se trata de una base de datos NoSQL que emplea JSON para almacenar los datos y JavaScript como lenguaje de consulta.

Lo que encontrarás en nuestra web

Se encontrará información, historia, guía de instalación y otras utilidades que permiten al usuario conocer más acerca de CouchDB y su funcionamiento como gestor de bases de datos noSQL.

Características de CouchDB almacena datos y cómo trabaja con ellos - Pt. 1

Las características más peculiares que tiene CouchDB es la facilidad con la que permite hacer replicaciones y la tolerancia a fallos, lo cual a día de hoy, mantener nuestro servicio online el mayor tiempo posible, es vital.CouchDB almacena los datos como "documentos", esto es, uno o más pares campo/valor expresados en JSON. Los valores de los campos pueden ser datos simples como cadenas de caracteres, números o fechas. Pero también se pueden usar listas ordenadas y vectores asociativos. Todos los documentos en una base de datos CouchDB tienen un identificador único y no requieren un esquema determinado.

Cómo CouchDB almacena datos y cómo trabaja con ellos - Semántica ACID - Pt. 2

CouchDB provee una semántica de atomicidad (propiedad que permite saber si algo se ha realizado o no), consistencia, aislamiento y durabilidad​. Lo hace implementando una forma de control de concurrencia multiversión, lo que significa que CouchDB puede manejar un gran número de lectores y escritores en paralelo, sin que surjan conflictos.

Cómo CouchDB almacena datos y cómo trabaja con ellos - Arquitectura distribuida con replicación - Pt. 3

CouchDB se diseñó teniendo en mente la replicación bidireccional (o sincronización) y la operación off-line. Eso significa que múltiples réplicas pueden tener cada una sus propias copias de los mismos datos, modificarlas y luego sincronizar esos cambios en un momento posterior.

Cómo CouchDB almacena datos y cómo trabaja con ellos - Interfaz REST - Pt. 4

Todos los ítems tienen una URI única que queda expuesta vía HTTP. REST usa los métodos HTTP POST, GET, PUT y DELETE para las cuatro operaciones básicas CRUD (Create, Read, Update, Delete) con todos los recursos.

Cómo CouchDB almacena datos y cómo trabaja con ellos - Operaciones de datos off-line - Pt. 5

CouchDB puede replicar datos a dispositivos (como teléfonos inteligentes) que pueden quedar offline y manejar automáticamente la sincronización de los datos cuando el dispositivo vuelve a estar en línea.
